Actions

Each category or type of action has a default named individual (specified in brackets in the list below) defined in the ontology. For more specitivity, the category can be subclassed, whereas the default named individual can be used to specify the type of action.

  • Collect (Collect Personal Data)
  • Erase (Erase Personal Data)
  • Read (Read Personal Data)
  • Share (Share Personal Data)
  • Store (Store Personal Data)
  • Use (Use Personal Data)
  • Write (Write Personal Data)
Status

Status denotes the state of consent (instance). These are categorised as valid/invalid for the purposes of processing. Each category has some named individuals defined in the ontology specified in the lists below.

Status valid for Processing

  • Explicitly Given
  • Given by Delegation
  • Implicitly Given

Status invalid for Processing

  • Expired
  • Invalidated
  • Not Given
  • Refused
  • Requested
  • Unknown
  • Withdrawn
